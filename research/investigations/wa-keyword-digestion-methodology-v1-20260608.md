# Keyword digestion — methodology (M01 evidence)

> **Investigation · v1 · 2026-06-08 · CC.** The researcher's three observations on the P1 keyword build
> ([wa-p1-keyword-rebuild-M01-v1](wa-p1-keyword-rebuild-M01-v1-20260608.md)) point at *how keywords are
> digested* — a methodology question, not a display one. This works each observation against the actual M01
> data and proposes a digestion rule. The keyword layer is a **structural input to L2 sub-grouping**, always
> validated by the verse read (verse meaning rules; keywords never decide alone).

---

## Observation 1 — "same keywords, different terms" → SYNONYMY → sub-group seed

Different lexemes carrying the same gloss is the *expected* shape of a word-cluster: many Hebrew/Greek terms
map onto overlapping meaning. On M01 the overlap is not random — it resolves into **~5 keyword-families**:

| Family (keyword signature) | Example terms |
|---|---|
| **fear / terror / dread** (core) | e.mah, chat, chit.tah, ma.gor, me.go.rah, pa.chad, ya.re, yir.ah, de.chal, mo.rah |
| **trembling / quaking** (physical response) | cha.rad, ra.gaz, rag.gaz, rog.zah, pa.lats, pal.la.tsut, ra.ad, re.tet, sa.ar |
| **dismay / alarm** (sudden fright) | ba.hal, be.hal, be.ha.lah, te.vah, cha.tat |
| **astonishment / amazement** (awe-adjacent) | thambos, ta.mah, tim.ma.hon, sham.mah, ek.thambos |
| **reverence / awe** (worship pole) | deos, fobos, *and the reverence sense of* ya.re / pa.chad |

**Digestion:** keyword **overlap is the mechanical seed for sub-group formation** (the old Phase-5 / new L2
sub-group step). The families above are candidate sub-groups *proposed* from keywords, then confirmed (or
re-cut) by reading the verses.

## Observation 2 — "some terms quite different… a descriptor?" → DIVERGENCE → role/relevance flag

The keyword-divergent terms are **not one kind** — divergence is the flag, the *disposition* is per-term:

| Divergent term | Keywords | What it actually is |
|---|---|---|
| H4035 me.gu.rah | fear, terror, **storehouse, granary** | **homonym** — two unrelated senses share the form |
| H4172B mo.ra | fear, **appoint**, terror | sense-intrusion from a neighbouring root |
| H3735 ke.ra | **distressed, grieved** | **cross-cluster** — sorrow/grief, not fear |
| H6125 a.qah | **pressure, oppression** | adjacent (distress), not fear |
| H1670 de.a.vah | **faintness, failure, mental energy** | adjacent (despair/faint), not fear |
| H4867 mish.bar | **wave, breaker, breaking** | **physical** term; fear only by metaphor (overwhelming) |
| H6178 a.ruts | **chasm, ravine, steep, slope**, dreadful | **place/physical**; "dreadful" is the thin fear thread |

So the researcher's "descriptor" intuition is the right *signal* — divergent keywords mean **scrutinise this
term's fit** — but the resolution splits four ways: **homonym-split · cross-cluster route · physical/metaphor
set-aside · genuine descriptor**. The keyword layer **auto-surfaces exactly the terms that need a relevance
decision** (this is the same surfacing the audit is meant to do — see [[feedback_audit_must_be_self_critical]]).

## Observation 3 — "focused vs large array" → POLYSEMY → sense-resolution priority + multi-sub-group

- **Tight arrays** (a.yom "terrible, dreadful"; e.mah "terror, dread"; de.chal "fear"; ra.hah "fear") =
  **mono-semic** — STEP gives one crisp sense; the term places **once, cleanly** (the S1 class).
- **Large arrays** (ya.re — 22 kw spanning *fear ↔ revere ↔ awe ↔ honour ↔ reverence ↔ terrify*; fobos — 14
  spanning *fear ↔ reverence ↔ awe ↔ astonishment*) = **polysemous head-terms**. Significance: a large array
  means the term (a) **needs per-verse sense-resolution** — and this is exactly where the **morph stem**
  decides it (ya.re Qal=fear / Niphal=be-feared-awesome / Piel=terrify, confirmed in
  [wa-step-morph-viability-M01-v1](wa-step-morph-viability-M01-v1-20260608.md)); and (b) **belongs in more
  than one sub-group** (ya.re sits in both *fear/terror* and *reverence/awe*).

**Caveat — array size is currently contaminated by source scope.** Some arrays come from the long *medium*
prose definition (glue words: "occurs only in the passive…"), others from a short gloss. **Normalise the
keyword source (gloss + top-level STEP senses, consistent across terms) before treating array size as pure
polysemy.** This is the P1 source-clean already flagged (concatenation artifacts + glue words).

## Proposed digestion rule (for L2 sub-grouping)

1. **Normalise** the keyword source scope (gloss + top-level senses) so size = real polysemy.
2. **Overlap → sub-group seeds** (the synonym families) — proposed, not fixed.
3. **Divergence → relevance/role flags** (homonym-split · cross-cluster · physical set-aside · descriptor).
4. **Polysemy (large array) → flag for morph sense-resolution + expect multi-sub-group membership.**
5. **Every seed/flag is validated by the verse read** — keywords are a structural input to L2, never the
   decider (the governing two principles: verse meaning rules; record everything).

This refines [[feedback_phase2_passa_emits_keywords]] (keyword analytics → sub-group input) and connects to
[[project_p2_l2_decision_architecture]] (overlap feeds sub-grouping; divergence = the S0/relevance + homonym
flags; polysemy = the S2 morph trigger).
