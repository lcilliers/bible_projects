## 1. What changed (the headline)

The study's **object** has changed. We **no longer identify, name, and describe individual characteristics** of the inner being. We now analyse its **movements, associations, interlocking, and emergence** — the inner being modelled as a **process / relational web**, read off **what each verse *does***, with patterns allowed to **emerge** rather than being sorted into a pre-set grid.

This was forced by the work: trying to sort verses into characteristics/units kept failing the same way (boundaries won't hold); a cold-read validation found ~70–85% of the prior analytical claims were *imposed*, not derived (the grid was relabelling fixed lemma tags); and a behavioural-science reckoning confirmed the inner being has no clean parts-anatomy. The unit-failure was *information about the subject*.

## 2. What is now CLOSED (stop doing these)

- **Naming/describing "characteristics" as the unit.** Characteristics are not the object anymore.
- **Object-type bins** (characteristic / state / expression / qualifier / identity / bivalent) as a classification grid to sort terms into.
- **Faculty as an ontology** ("what faculty *is* this term / does the inner being *have*"). The intrinsic-faculty field is retired.
- **The tier catalogue (T0–T7) as a classification grid**, and **"logical units"** as the organising move.
- **Pole-pairs / §10 object-kinds / per-cluster "characteristic synthesis"** as the framing.
- Treating **clusters as meanings** (they are a porous workspace only) or **frequency as a gate** (count never decides).

## 3. The NEW method (do these)

**Decomposition — per verse, record WHAT THE WORD DOES** (the functional/relational edges), each an *observation* with three states (resolved / NONE-silent / UNRESOLVED) and a citation:
- antecedent/cause · operation/how · object/target (+ observed object-kind) · manner/intensity · produces/effect · immediate-response · transition/becomes · relational-web/binding · direction.
- **Faculty → the observable:** *"does this verse mention or address a faculty/seat, and which"* (heart/soul/spirit/mind/conscience/understanding/will…) — presence, not essence.
- **Discovery-lookout (mandatory, every verse):** *"what does this verse state or imply about the inner being that the current considerations do NOT capture?"* → flag it (so the schema grows from the material). A verse with nothing records "discovery: none."
- **Original-language grounded** (lemma/morph, never the English string). **No imposed categories.**

**Assembly — let patterns emerge as MOVEMENTS** (recurring functional shapes: antecedent→operation→object→effect→transition), the **movement (not the term) as the unit**, across a four-level cascade: in-verse → collection-of-similar-verses → cluster → cross-cluster.
- **Measurement informs, never decides** (count/co-occurrence reveal candidates, never gate existence/inclusion/exclusion/validity).
- **A movement of one is a movement** (attestation includes · validity excludes · frequency does neither).
- **Discard↔include balance is calibrated in situ** — bias toward surfacing/keeping while unknowns are immature.
- **Everything traceable** — every finding walks back to its verses.

## 4. What to UPDATE in your (Claude AI) memory / project-knowledge / history

- **Supersede** any stored instruction that says the task is to classify/name characteristics, assign object-types, derive faculties as essences, or sort into tiers/units. Mark those as **legacy (pre-2026-06-25)**.
- **Adopt as the live method** the two reset specs (titles below) + the milestone review.
- **Mark all prior analytical outputs as LEGACY to be revisited** — all lexical analysis, all M01–M11 "completed" cluster analyses, all in-progress work. They were produced under the closed framing.
- **Keep as valid:** the original-language measure layer, the base per-verse sense-readings, the corpus — and the *lessons* (observe-don't-impose; ground in per-verse-varying evidence; measurement informs never decides; plausibility ≠ truth).
- When asked to analyse a verse, produce the **functional edges + the discovery-lookout**, not a characteristic/object-type label.
