## 3. Key Design Decisions

### D1 — T-AR confirmed by fellowship run
The fellowship Stage 1 run validated the Analysis Readiness instruction. T-AR is now CONFIRMED.

### D2 — Six-pass model replaced by three-stage model
v5.0 Stage 2 organised analysis into six structured thematic passes (meaning, divine, verse annotations, somatic, language, correlation). This instruction replaces that with:
- Stage 2a: free-form comprehensive narrative — everything the data says, no imposed structure
- Stage 2b: catalogue Q&A partitioning — catalogue drives what is asked; Stage 2a provides the evidence
- Stage 2c: six reader-facing chapters — analytic word output serving Session C and Session D

This is not a renaming of six passes. It is a different analytical approach. Stage 2a allows Claude AI to read everything before structuring anything. Stage 2b uses the catalogue as the organising principle rather than six thematic lenses. Stage 2c produces reader-facing chapters, not an internal brief.

### D3 — Analytical Brief retired
v5.0 produced an Analytical Brief (nine-section internal document) as the Stage 2 handoff. This is replaced by Stage 2c's Analytic Word Output (six chapters, reader-facing). The new output serves both the Session C production function and the Session D handoff function more effectively.

### D4 — Database writes occur at Stage 2b close, not at each pass close
v5.0: every pass closed with a confirmed database write. New: Stage 2a produces no database writes. Stage 2b constructs and applies all Type (b) patches after Q&A pairs are formed. This preserves the separation between free-form observation and structured persistence.

### D5 — Characteristic-perspective grouping model applied at reading level
Stage 2a Unit 7 specifies that every anchor verse is read with its group description as the interpretive lens. This operationalises GR-PROG-006 at the verse-reading level — the group description names what inner-being characteristic the verse engages.

### D6 — Stage 2a reading sequence is defined and ordered
Nine reading units in a fixed sequence: registry overview → XREF terms → OWNER terms lexical → group landscape → correlation signals → existing SD pointers and findings → anchor verse reading → thin-evidence flags → existing findings review. Correlation signals are read before anchor verses so the cross-registry lens is calibrated before verse-by-verse work. This is a key improvement over v5.0 where cross-registry context came last.

### D7 — NOT ANSWERED vs NOT APPLICABLE distinction is critical
Stage 2b introduced a four-disposition system for catalogue questions: ANSWERED, PARTIALLY ANSWERED, NOT ANSWERED, NOT APPLICABLE. The critical distinction is between NOT ANSWERED (Stage 2a should have addressed this and did not — triggers Stage 2a reopening) and NOT APPLICABLE (the characteristic genuinely does not engage with this question's domain). The instruction assigns NOT APPLICABLE conservatively to prevent analytical gaps being mislabelled.

### D8 — Three integrity rules added: SB-25, SB-26, SB-27
- SB-25: Stage 2a observations log is fixed after sign-off. Stage 2a reopening adds new units sequentially — does not modify existing entries.
- SB-26: Stage 2b draws only from Stage 2a. No new analysis introduced.
- SB-27: Stage 2c draws only from Stage 2b Q&A pairs and SD pointer accumulator. No new analysis introduced.

These three rules enforce the unidirectional flow: Stage 2a produces the evidence; Stage 2b structures it; Stage 2c presents it. Each stage is contained.
