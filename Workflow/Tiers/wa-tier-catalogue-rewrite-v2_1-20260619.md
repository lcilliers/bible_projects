## Conventions (read once)

1. **8 areas kept.** T0–T7 titles and sub-tier scope/order unchanged.
2. **Disposition per code.** Every original code carries exactly one disposition:
   - **`keep`** — code stays active (`deleted=0`); `question_text` updated to the rewritten form below.
   - **`obsolete`** — code folded into a named **primary** (`deleted=1`); its distinct content is preserved in the primary's **`absorbs:`** line. Reversible soft-delete — nothing is lost, the merge can be undone.
3. **No lost detail.** Each consolidating (primary) question names, in an `absorbs:` line, every distinct prompt it now carries.
4. **De-biased phrasing.** Questions ask *what the evidence shows / which items are present and how they behave* — never *"does it have property P?"* or *"what does it reveal about Y?"* framed to expect a yes. Expected-answer verbs (**equip, receive, give, enable/deepen/bypass/impair**) removed.
5. **Silence is first-class.** Every kept question can record absence. The dedicated *"note silence explicitly"* rows are therefore redundant and become `obsolete`.
6. **List-grids collapse.** Where a sub-tier set asks the same question of a predefined list (the six constitutional levels), it becomes one open question with the list as the **answer-set / completeness audit**, plus one synthesis question. Listed items are retained as the audit so scope cannot shrink.
7. **CC update model.** This document is the decision source. CC derives two operations per code from it: a text-update for `keep` codes and a soft-delete for `obsolete` codes. The JSON patch is CC's to build (GR-PROG-005).

> **Net across T0–T7:** 173 original codes → **126 `keep`** (active, rewritten text) + **47 `obsolete`** (folded with pointers, soft-deletable). No prompt detail dropped; every question de-biased.
