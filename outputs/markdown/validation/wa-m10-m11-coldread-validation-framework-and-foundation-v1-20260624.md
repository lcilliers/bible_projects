# M10 + M11 cold-read validation — framework, foundational findings, pilot

- **File:** wa-m10-m11-coldread-validation-framework-and-foundation-v1-20260624.md · **v1 · 2026-06-24 · Author:** Claude Code.
- **Mandate (researcher, 2026-06-24):** the programme's recurring failure is the AI reflex to **get a concept, then make the evidence fit it** (eisegesis), instead of **gathering the evidence and reflecting back what it says** (induction). Thoroughly revalidate **all M10 and M11 work** to stamp this out. The corrective method is the **cold read against the lexical**.
- **Trusted base:** the **lexical** is the evidence base and is low-error (researcher: "minimal errors I'm aware of in the lexical"). Validation is done **against the lexical, not the root verse text.** If validating a claim **requires the root verse**, that is itself a finding — **the lexical is defective for that occurrence** (logged, not relied upon).

## §1. Method — cold read against the lexical (anti-confirmation)

1. **Induction, not deduction.** For each unit, read the **aggregate per-verse lexical evidence** and state what it says *before* looking at the claim/label. The label is examined last.
2. **Evidence → claim order.** Never claim→evidence (that harvests confirmations).
3. **Null default.** Every claim is `IMPOSED`/`OVERREACH` until the per-verse evidence earns `SUPPORTED`. A pass that finds little imposition is itself suspect.
4. **Use only per-verse evidence for per-verse claims** (see §2): `sense · object · object_type · experiencer · how(where present) · valence(partly) · co-occurring terms`. **`faculty`, `type`, `lemma_meaning` are lemma-constants** — a per-verse claim resting on them is imposition unless the per-verse fields independently carry it.
5. **Root-verse recourse = logged lexical defect.** If the lexical can't settle a claim and the root verse is needed, record it as a `LEXICAL-GAP` (which field was empty/insufficient), don't lean on it to win the argument.
6. **The §10 grid is on trial too.** The object-kind typology is not assumed; the validation may conclude it is an overlay the verses don't support.

**Verdict codes:** `SUPPORTED` (per-verse evidence carries it) · `OVERREACH` (claim exceeds the evidence) · `IMPOSED` (label not in the evidence; from the frame) · `CONTRADICTED` (evidence says otherwise) · `TERM-CONSTANT` (claim reads per-verse meaning from faculty/type) · `LEXICAL-GAP` (needed the root verse → lexical defect).

## §2. FOUNDATIONAL FINDING — which lexical fields are per-verse vs lemma-constant

Measured across every focus occurrence (M10 = 2,193 occ / 86 terms; M11 = 288 occ / 15 terms): **% of terms whose field value varies across that term's occurrences.**

| Field | M10 varies | M11 varies | Verdict |
|---|---|---|---|
| **faculty** | **0%** | **0%** | **LEMMA-CONSTANT** — never per-verse |
| **lemma_meaning** | 0% | 0% | LEMMA-CONSTANT (definitional) |
| **type** | 14% | **0%** | **mostly LEMMA-CONSTANT** |
| valence | 36% | 67% | partly per-verse |
| how | 63% | **13%** | per-verse but **sparse** (M11 = 8/288 occ) |
| object_type | 69% | 73% | per-verse ✓ |
| object | 70% | 73% | per-verse ✓ |
| sense | 77% | 80% | per-verse ✓ |
| experiencer | 76% | 80% | per-verse ✓ |

**Consequences (mechanical, before any reading):**
- **Every "this verse is faculty-X-seated" claim is invalid** — faculty is a lemma-constant (cha.ta = moral_evaluation in all 213 verses; a.von = none in all 162; na.cham = affect in all 34). The whole class of per-verse seatedness claims (M10/M11/M12, incl. U4 "innocence conscience-seated 100%") reads verse-meaning out of a fixed tag.
- **Object-kind keyed to `type` is a relabel** — in M11 `type` *never* varies, so "reconciliation = action", "atonement = action" are lemma tags, not verse readings; sorting them into §10 kinds by type imports the grid.
- **`how` is too sparse to carry operational claims**, especially in M11 (3% coverage). Operational/"how sin works per verse" claims rest mostly on absent data → expect `LEXICAL-GAP`.
- **The legitimate per-verse evidence is `sense · object · object_type · experiencer` (+ valence).** Any unit's object-kind must be re-derived from *what the term relates to / does* in the verse, via these — not from faculty/type.

> Distinction worth keeping: **co-seatedness** (which *other* terms appear in the verse — nephesh/lev/conscience) IS per-verse and legitimate; the **faculty tag** is not. Earlier reads conflated the two.

## §G — Bias-guard on the validation itself
- The validator (incl. any sub-agent) is Claude and carries the same reflex; the discipline (induction-first, null-default, faculty/type excluded, goal = *find* imposition) and the **mechanical per-verse/constant split (§2)** are the guards.
