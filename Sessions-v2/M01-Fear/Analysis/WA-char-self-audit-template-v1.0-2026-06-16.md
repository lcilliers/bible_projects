## The standing rule

Before any per-characteristic analysis is presented, Claude AI runs a self-audit that proves **every lexical field of every verse in the characteristic's set has been examined** — each field yields either a value reported or a silence noted. Nothing is presented until the self-audit table (§3) is complete. The completeness check is Claude AI's own responsibility, run proactively — not deferred to review.

---

## 1. Field-completeness — every field gets a value or a noted silence

For each characteristic, account for **all** lexical fields, never a subset:

`sense · lemma_meaning · type · valence · object · object_type · divine_involvement · cause · location · faculty · compound · immediate_response · experiencer · mode · morph/stem · intensity · how · relational · origin`

Plus the term layer (`strong · translit · gloss · language`) and the verse layer (`reference · testament · verse_text`).

- A field with no data → state the **silence** explicitly (and whether it is a true silence or **excluded-by-design**, e.g. purpose/typology have no lexical basis).
- A field that is empty in the lexical block but carried elsewhere (e.g. `mode` empty but grammar in `morph/stem`) → read it from where it lives.

## 2. Every angle — each field examined more than one way

Looking "from all angles" means each populated field is read along the relevant cuts, not merely noted as present:

- **Distribution**, not just presence — report the counts/spread.
- **Cross-tabulation** — the field against others where it sharpens meaning (e.g. valence × object_type; stem × valence; type × testament).
- **Direction** — for relational fields, every direction (e.g. `divine_involvement` object vs agent vs giver vs addressee).
- **Within-term vs across-term** — does the pattern hold per lemma, or only in aggregate?
- **By testament / language** — OT/NT, Hebrew/Aramaic/Greek splits.
- **Multi-value cases** — both the de-duplicated count and the multi-value occurrences themselves (faculty blends, multi-location).
- **Boundary / contrast** — the forbidden/sinful/negated cases as definitional evidence, not noise.

## 3. Beyond the lexical fields

- **`compound`** — always reported per characteristic: co-term nexuses routed to the tiers they inform (T1/T3/T5/T6); genuine other-characteristic partners parked for T7 while T7 is held.
- **`verse_report` morphology** — the grammatical axis (stem, imperatives, participles) as semantic evidence.
- **Integrity / anomalies** — duplicates, homonym/sense noise, count mismatches, registered-form pairs — surfaced, not silently passed.
- **Verse-text grounding** — claims checked against the actual `verse_text`, not only the field values.

## 4. The audit gate (produced before presenting)

A field-completeness table is built first, e.g.:

| Field | Fill (of N) | Addressed in / angle taken | Status (mined / cited / noted-silence / excluded-by-design) |

The analysis is presented only once this table accounts for every field. The table travels with the analysis (as in C1 v1.3 §9).
