# Sessions-v2 — per-cluster working tree (the v2 home)

> **Created 2026-06-05.** The fresh, authoritative home for **all new cluster work**, one folder per
> cluster. Supersedes the scattered old layout for *going-forward* output; the old folders stay intact.

## The approach

Work the clusters **from M01 upward, one at a time**, making **each cluster the best it can be** against
the settled foundations (`Workflow/methodology/wa-study-foundations.md`) and the cluster audit aspect spec
(`Workflow/methodology/wa-cluster-audit-aspect-spec-v1-20260601.md`). **Clusters will be re-done where
needed** — a prior "Analysis Completed" status is not a reason to skip; the bar is now "best it can be",
not "gate cleared".

Sequencing (see memory `project_audit_sequencing_clusters_then_meaning`): bank a good set of clusters to
**clean** status via the structural audit (Groups A–D) first; the **meaning/keyword-quality audit**
(Group E) runs as a later dedicated phase.

## Structure

- One stub folder per cluster: **`{CODE}-{Name}`** — e.g. `M01-Fear`, `M10b-Wickedness`, `M38-Salvation`.
- **48 folders** = 46 M-clusters (M01–M46, gaps at M32/M40, plus the M10b/M10c splits) + `FLAG-Flag`
  (REVIEW bucket) + `T2-Supplementary` (SUPPLEMENTARY bucket).
- **All output relating to a cluster is saved in its folder here.** File naming follows
  `docs/file-organisation-rules.md` — cluster files: `wa-cluster-{CODE}-{kind}-v{n}-{YYYYMMDD}.{ext}`
  (lowercase prefix, CAPS code).

## Cross-reference to the old tree (read-only)

The old folders — `Sessions/` (incl. `Sessions/Session_Clusters/{CODE}/`), and the per-stage
`Session_A…D` folders — **stay intact and are used only as cross-reference**. Do **not** write new cluster
output there. The database remains the single authoritative record of findings regardless of folder.
