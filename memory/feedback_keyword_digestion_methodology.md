**Researcher observations on the P1 keyword build (2026-06-08), worked against M01 data
(`research/investigations/wa-keyword-digestion-methodology-v1-20260608.md`).** These are *methodology* — how
the keyword layer is digested into L2 sub-grouping.

**The three signals:**
1. **Same keywords / different terms = SYNONYMY → sub-group seed.** Not random: M01 resolves into ~5 keyword
   families (fear/terror/dread · trembling/quaking · dismay/alarm · astonishment/amazement · reverence/awe).
   Keyword **overlap is the mechanical seed for sub-group formation** (old Phase 5 / new L2).
2. **Divergent keywords = a relevance/ROLE flag (not one kind).** Disposition is per-term: **homonym-split**
   (me.gu.rah = fear + storehouse/granary), **cross-cluster** (ke.ra = grief→sorrow), **physical/metaphor
   set-aside** (mish.bar = wave/breaker; a.ruts = ravine), or genuine **descriptor**. Divergence
   auto-surfaces the terms whose fit needs a decision (same job as the self-critical audit).
3. **Array size = POLYSEMY.** Tight array = mono-semic, places once cleanly (S1). Large array (ya.re: 22 kw,
   fear↔revere↔awe↔honour↔terrify) = polysemous head-term → (a) **needs per-verse morph sense-resolution**
   (Qal/Niphal/Piel), (b) **belongs in >1 sub-group**.

**Prerequisite:** array size is currently contaminated by inconsistent source scope (full medium prose vs
short gloss). **Normalise the keyword source (gloss + top-level senses) before treating size as polysemy** —
this is the P1 source-clean (concatenation artifacts + glue words).

**Digestion rule:** normalise → overlap=sub-group seeds → divergence=relevance/role flags → large-array=morph
sense-resolution + multi-sub-group → **every seed/flag validated by the verse read** (keywords are structural
input to L2, never the decider). Refines [[feedback_phase2_passa_emits_keywords]]; connects to
[[project_p2_l2_decision_architecture]] (overlap→sub-grouping; divergence→S0/homonym; polysemy→S2 morph).
