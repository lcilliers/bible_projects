**Roll-out strategy for the verse-read = meaning layer (2026-06-09).** Researcher's call after seeing M01:
the meanings are "incredibly rich, far above prior verse work, none disagreed with."

**Be patient: accumulate clusters through the full verse-meaning cycle BEFORE distilling.** Do not build the
characteristic-behaviour distillation / synthesis layer yet ([[wa-characteristic-distillation-design-v1]] is
the sketch, on hold). Reason: avoid prematurely freezing a synthesis shape while the roll-up UNIT is still
open ([[feedback_ontology_typed_relationships]] — cluster may not be the unit).

**The named risk: losing the value to VOLUME.** ~2,750 paragraphs for M01 alone; ×46 clusters is an unreadable
mountain if treated as prose.

**Why the value is safe:** every paragraph is mirrored as ~14 individually-tagged findings (origin/faculty/
effect/response/relational/… each its own row), so the corpus is digestible AS DATA — aggregate, filter,
group, distil at full fidelity anytime, without re-reading. Deferring distillation loses nothing.

**The safeguard (cheap, per cluster, as each completes):** a standard lightweight GATE — (1) mechanical tier
profile (instant, read-only), (2) flag rate, (3) a few spot-checked sample paragraphs. Catches method drift
before it spreads corpus-wide; leaves a 1-page digestible headline per cluster instead of a backlog. This is
the M01 review trimmed + standardised; produce it automatically per cluster.

**Cost trajectory:** ~46 clusters, trending cheaper as cross-cluster fan-out pre-seeds each next cluster
(M01 ~$14/4.5h; report actual per cluster). Status: M15 running ([[project_l2_verse_read_meaning_live]]).
