**Checkpoint asked by researcher (2026-06-08):** stick with V3_2 cluster-based analysis and build on it, or
has the L1 corpus work pointed to a different approach? Honest weighing in
`research/investigations/wa-method-checkpoint-does-l1-change-the-approach-v1-20260608.md`.

**Verdict: build on V3_2 — it evolves, it is not replaced.** Every L1 finding either *confirms* V3_2's spine
(verse as unit; *a finding for every verse*) or *adds an attribute* to the finding — none *breaks* it.
- **Enrich/confirm:** morph backfill (sense + thing-type lever) · typed keyword map · co-occurrence (confirms
  cluster = scaffolding) · three link-classes (structures the cross-cluster layer) · ontology reframe
  (the finding becomes a typed relationship) · verse-coverage (= V3_2's own L1 verse-centricity).

**The one real shift (don't pretend nothing changed):** the **primary output moves** from *characterise each
cluster* (L2 sub-groups→L6 characteristic→L7 synthesis) to a **typed-relationship finding for every verse**.
The **cluster demotes to a work-batch + roll-up**; the **relationship layer (was L8, the tail) becomes
central**; findings gain **thing-type (status/action/quality) + relationship-type + effect**; the morph
**stem = the operation-mode** ("method A→Y"); **meaningful-absence** (KIN-not-met / structural holes) becomes
a finding-type; **verse-coverage discipline** (a cross-cluster verse analysed once, shared; cluster = entry
order not boundary, [[feedback_l1_l2_is_multi_angle_report_then_synthesise]]).

**Why not a clean-sheet verse-first rebuild:** it converges with the evolution (both make the verse the unit
and the typed relationship the finding); the only difference is cluster-as-processing-batch (keep) vs
derived-view (drop). Rebuild pays twice ([[feedback_no_rework_paid_twice]]) — the DB findings model,
cross-cluster link tables and dual-cluster-finding handling already exist; the cluster roll-up is a wanted
product. So keep the batch, change the emphasis.

**Open question to settle WHILE building (not pre-decided):** does the V3_2 upper machinery (L2 sub-groups,
L6 characteristic assessment) still earn its place, or become **roll-up views over the verse-level findings**
rather than structure imposed before the read? Risk = forcing cluster structure the data resists
([[feedback_no_forced_structure_audit_surfaces_analysis_compensates]]). **Recommendation:** keep L1 + the
finding as record; treat L2–L7 as roll-ups built from verse findings; **prove on Fear first.** Connects to
[[feedback_ontology_typed_relationships]] and [[project_cross_cluster_three_link_classes]].
