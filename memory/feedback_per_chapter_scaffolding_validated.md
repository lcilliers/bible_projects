The per-chapter Session C scaffolding produces fluent integrated prose when used as intended. Validated 2026-05-29 on M38 chapter 2: researcher response was "better than M11 — more direct, less padding, feels founded, grounded, supported by appropriate verses."

**Why the scaffolding works:**
- Topic sentences carry the argument (forced by chapter instruction structure)
- Each section is bounded — no temptation to drift into other chapters' territory
- Required-cite verses anchor each section at strongest points
- Distinguishing sentences force positioning against neighbours
- Reverse audit (§10) + self-review (§11) catch repetition, hedging, padding, vocabulary drift before save

**Why the M38 essay v1 failed:**
The single-pass whole-essay approach bypassed the scaffolding. CC drafted from a 189-finding synthesis dump in one go, holding too much in mind at once, producing prose that read as sentence-by-sentence assembly. The findings were fine; the process was wrong.

**How to apply:**
- Phase E authoring = one chapter per CC response. Never batch chapters.
- Each chapter loads exactly three documents: chapter input + chapter instruction + style guide.
- Reverse audit + self-review applied before save, every chapter.
- No vision document, consistency pass, or integration pass needed at this scale — the existing scaffolding is sufficient.
- v3_0 owns Phase A–D. Phase E uses the Session C process exactly as it ran for the 14 catch-up essays and M01/M09/M11.
