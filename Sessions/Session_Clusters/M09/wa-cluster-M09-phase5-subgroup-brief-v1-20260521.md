## THE PRIMARY OBJECTIVE — sub-groups represent characteristics (§8.0)

**This is the structural objective.** Read it carefully — it shapes every step that follows.

1. **A sub-group represents a characteristic** — a single inner-being faculty/state. The default is **1 sub-group : 1 characteristic**.
2. **Volume-split is the documented exception.** When a single characteristic's verse corpus would exceed the §8.6 distribution gate (40% of substantive verses = **>44 verses** for M09), split it into multiple sub-groups by a documented split-axis. The characteristic identity persists across the splits.
3. **A sub-group serving two characteristics (SPLIT_SUBGROUP) is rare** — only when the same lexical sub-group contains different VCG registers serving different characteristics. Flag any such case at Phase 5.
4. **Cross-register flags from Phase 3 inform characteristic identification.** A term carrying a cross-register flag may suggest a characteristic that the cluster's primary register doesn't fully cover.

## Process

### Step 1 — Identify the characteristics

Read across the meaning corpora (§2 of the constitution report) and name the distinct **inner-being faculties / states** M09's evidence expresses. Aim for an analytically clean list — **typically 3 to 6 characteristics** for a cluster of this scale (109 verses, 17 terms).

A characteristic is a single answer to: *"what specific inner-being faculty or state does this body of verses evidence?"* For M09, candidate characteristics suggested by the Phase 3 patterns (this is suggestive, not prescriptive — derive your list from the meanings):

- **Humility as inner disposition** — the core *tapeinos / tapeinofrosunē / tsa.nu.a / ka.na voluntary-humbling* register where the inner self lowers itself
- **Contrition / brokenness of spirit** — *dak.ka*; acute form following genuine moral failure
- **Submission of will to authority** — *hupakoē / hupakouō / hupotagē / yiq.qe.hah*; inner disposition that yields
- **Meekness / gentleness / restraint** — *metriopatheō* + the gentleness dimension of *tapeinos* + *okneō* (appropriate restraint)
- **Dignity as grounded worth** — *semnotēs*; moral gravity without self-display
- **Willing-heartedness** — *na.div*; the freely-moved will

Refine, combine, or split these from the actual meaning evidence. The cross-register flags are reminders that these dynamics belong in M09 with the flag noting the other-register dimension.

### Step 2 — Map each characteristic to its evidence

For each characteristic, note which **terms' verses primarily evidence it**. Some terms will be primary carriers; others will be secondary (their corpus splits across characteristics, or they're flagged cross-register).

### Step 3 — Design sub-groups to carry the characteristics

- **Default**: one sub-group per characteristic. Use natural codes (`M09-A`, `M09-B`, ...).
- **Volume-split**: if a characteristic's verse evidence would exceed **>44 verses** (40% of 109), split that characteristic into multiple sub-groups by a named split-axis.
- Each sub-group's `core_description` explicitly names: (a) the characteristic it represents, (b) the split-axis if it's part of a volume-split, (c) the inner-being content from the meanings.

**Note on M09's small corpus:** 109 verses across an estimated 4-6 characteristics gives an average of ~20 verses per sub-group. Distribution is unlikely to trip the §8.6 gate unless one characteristic dominates excessively. If ka.na's 18 verses + tapein-family's ~30 verses cluster into one "humility-disposition" sub-group, that's still within gate (~48 / 109 = 44%). Slightly above — would warrant a volume-split, e.g. by voluntary-humbling vs imposed-lowliness OR by Greek-tapein vs Hebrew-ka.na.

### Step 4 — Assign every verse to exactly one sub-group

For each of the 109 Pass A meanings, record the sub-group assignment. Mapping form: `{vc_id: subgroup_code}` OR verse-reference-based with per-term split rules (matches M07/M08 pattern). Output as JSON for CC's Phase 6 apply.

### Step 5 — BOUNDARY sub-group (likely empty)

Phase 3 produced **0 BOUNDARY verdicts**. No M09-BOUNDARY sub-group is needed unless Phase 5 design surfaces a fresh BOUNDARY question. If it does, follow §8.4.1 strictly (BOUNDARY carries only verses of BOUNDARY-verdict terms).

### Step 6 — Flag SPLIT_SUBGROUP cases

If any sub-group on closer inspection contains VCG-level registers serving different characteristics (rare), flag it in the output with the analytical question.

### Step 7 — Multi-faceted terms

Some terms' verses may span multiple sub-groups (e.g., tapeinos has both gentleness and humility dimensions). Record primary + secondary assignments per term in the mapping.

## Discipline

1. **Read every verse-meaning** before designing sub-groups. No shortcuts from gloss or transliteration.
2. **Characteristic-first design** (§8.0). Don't cluster by meaning-similarity until you've named the characteristics the cluster expresses.
3. **The 40% distribution gate is a volume-split trigger, not a fail signal.** If a characteristic carries >44 of M09's 109 substantive verses, split it into multiple sub-groups with a named axis.
4. **Cross-register flags travel forward.** When a flagged term lands in a sub-group, the flag is preserved in the rationale for Phase 7/9 consumption.
5. **No BOUNDARY parking** (§8.4.1). All 109 substantive verses route to characteristic sub-groups (no BOUNDARY-verdict terms to park).
6. **Luk 3:5 G5013 tapeinoō**: route to its natural M09-A1 (or wherever tapeinoō verses land) for now; the set_aside happens at Phase 5.5 via CC patch.
