# M01 Phase 6 — reminder before starting the VCG reconciliation pass

**Audience:** Claude AI
**Source:** [wa-sessionb-cluster-instruction-v1_13-20260514.md §9.3 + §2.1 + §2.3](../../Workflow/Instructions/wa-sessionb-cluster-instruction-v1_13-20260514.md)
**Reason for this note:** in earlier clusters Phase 6 Pass A was skipped or short-circuited — readings were summarised rather than per-verse-recorded, and inherited VCG labels biased the design. v1_13 §2.3 and §9.3 were tightened specifically to prevent this. This is the reminder before M01 Phase 6 begins.

---

## The non-negotiable in Phase 6 Pass A

> **Read every non-set-aside verse in the sub-group. For each verse, write to the obslog:**
>
> `{verse_reference} | {term} | one-line plain-English meaning of what the verse says about the characteristic in this verse's context`

One line, per verse, written **at the moment of reading the verse** — not reconstructed afterwards from a summary, not derived from the inherited VCG label.

## Three rules that always apply, called out explicitly here

### 1. Write-on-discovery (§2.1)

If a verse's meaning has not been written to the obslog before you move to the next verse, it is not written. Do not summarise at the end of a reading block and back-fill — that is the failure mode.

### 2. Inherited-structure contamination guard (§2.3)

The cluster carries inherited VCGs from contributor registries — for M01 that is 131 VCGs from 17 registries (or whatever the post-Phase-4 grouped report shows). These VCGs and their existing `context_description` text are **candidate evidence only**, not the answer. Treat them as background while you read; do not let the inherited label shape what you write as the verse's meaning. Pass A must produce a meaning record that could have been written if no inherited VCGs existed.

### 3. Three-pass order — never interleaved (§9.3)

- **Pass A** = read + record meaning, one line per verse. **Do not** name VCGs during Pass A.
- **Pass B** = group the meanings into provisional VCGs, designate one anchor per VCG.
- **Pass C** = reconcile against existing VCGs (KEEP / REFINE / SPLIT / MERGE / OBSOLETE / NEW).

If you find yourself sketching a VCG before Pass A is complete, stop and finish Pass A.

## The specific failure pattern v1_13 §2.3 was written to stop

> "the pattern that produced the failure in M15's first attempts: AI read the existing VCG labels and shaped its analysis to fit them."

The corrective is: **read the verses first; write what each verse means in its own terms; design groupings from those meanings; only THEN compare to the existing structure**.

## What Pass A's output should look like (illustration)

```
Job 12:5  | H7600 sha.a.nan  | The man at ease scorns the one whose foot is slipping — complacent inner posture mocks the vulnerable
2Ki 19:28 | H7600 sha.a.nan  | God names Sennacherib's complacent rage — divine response is judgement on the at-ease boast
Amo 6:1   | H7600 sha.a.nan  | Woe to those at ease in Zion — prophetic indictment of complacent security as inner failure
```

Three columns. One line per verse. Plain English. Grounded in the specific verse, not the cluster's general theme.

## If you have already done a partial Pass A and want to verify it counts

Test each entry against these:

- Does the line name **this verse's** content (not a generic statement about the term)?
- Could a reader who can't see the original verse approximately reconstruct what the verse says from your line?
- Was the line written **before** any VCG was named for this verse?

If yes to all three: keep. If any no: re-read and re-write.
