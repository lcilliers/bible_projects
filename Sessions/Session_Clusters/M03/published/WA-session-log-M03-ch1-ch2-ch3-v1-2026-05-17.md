## 3. Session failure — instruction compliance

### What happened
The researcher identified that chapter writing was being conducted in a single session accumulating all chapters, rather than in isolated per-chapter sessions as the instruction specifies. Chapter 3 was partially written before the session was terminated.

### What the instruction specifies
The shared style method document (§8) states: *"Each chapter is a standalone unit of prose. Other chapters of the study will be written in separate runs — you write only the chapter you are given."* The chapter instruction architecture loads: (1) the shared style method, (2) that chapter's own instruction, and (3) that chapter's own input file — per run, in isolation.

### What went wrong
All three chapter inputs were loaded in a single session. Chapters 1 and 2 were written sequentially within that session. Chapter 3 was begun before the researcher terminated the session.

### Impact
- Ch1 and Ch2 outputs are complete and appear correctly scoped. The content was written per-chapter sequentially and no cross-chapter content contamination is evident on inspection. However, these were produced in violation of the run-isolation architecture.
- Ch3 partial output (`wa-cluster-M03-ch3-draft-v1-2026-05-17.md`) is incomplete and must not be used.
- The researcher's instruction is that each chapter must be introduced in isolation in a separate session.

### Correct procedure for subsequent chapters
Each chapter is a separate session load:
1. Load shared style method.
2. Load chapter-specific instruction.
3. Load that chapter's input file only.
4. Write, audit, review, return.
5. No other chapter inputs are loaded in the same session.
